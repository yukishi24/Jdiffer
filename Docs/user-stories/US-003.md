# US-03: 比較条件プリセット機能

## User Story

- **As a**: 運用担当者  
- **I want**: 比較項目の設定をプリセットとして利用したい  
- **So that**: 毎回同じ条件を設定する手間を省き、作業を効率化したい  

---

## Background / Context

- 差分確認では、特定の項目のみを確認する運用が存在する  
- 毎回手動で設定するのは手間であり、設定ミスの原因となる  
- 定型的な確認作業の効率化が求められている  

---

## Acceptance Criteria (High-level)
※ 実装に依存しない、ユーザー視点の完了条件を書く
- [ ] ユーザーが比較対象項目を選択できること  
- [ ] 選択した条件をプリセットとして保存できること  
- [ ] 保存済みプリセットを選択して比較を実行できること  
- [ ] プリセットが存在しない場合でも通常比較が行えること  
- [ ] 保存・読み込み失敗時に適切なエラーが表示されること  

---

## Constraints / Non-goals（Optional）
※ 今回「やらないこと」「対象外」を明示する
- やらないこと：
  - プリセットのユーザー間共有機能
- 技術的制約：
  - 単一ユーザー利用を前提
- 将来対応予定：
  - プリセットのエクスポート／インポート
  - 組織内共有機能
